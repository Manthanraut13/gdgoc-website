Typography Scale

Three-font system: Syne for display authority, DM Sans for readable body, JetBrains Mono for code/terminal authenticity. Each has a distinct role — no overlap.

Font Families — Rationale

Display

Syne

Sharp, wide, modern. Feels technical and confident. Used for all headings. Its geometric character references engineering precision.

Body

DM Sans

Highly readable at small sizes. Clean, humanist structure. Friendly but professional — ideal for descriptions, cards, and nav. Superior x-height on mobile.

Mono / Code

JetBrains Mono

The developer's font of choice. Used for terminal components, code labels, section identifiers, and any element signalling "dev-native" identity.

01 · 06

Animation Principles

Motion should feel like a developer pushed an instruction — precise, immediate, purposeful. No decorative animations. Every motion communicates state or guides attention.

⚡

Scroll Reveal

Elements enter on scroll with a subtle upward slide + fade. Stagger delay of 80ms between items in a grid to create reading flow.

translateY(24px)→0 + opacity 0→1 duration: 500ms easing: cubic-bezier(0.16,1,0.3,1) delay: 80ms stagger
🖱️

Card Hover

Cards lift 4px and reveal a colored shadow on hover. Border transitions from dark to brand color. Inner content may shift slightly.

transform: translateY(-4px) box-shadow: --s-2 or --s-blue border-color: var(--g-blue) transition: 200ms ease-out
🔢

Count-Up Stats

Impact numbers animate from 0 to final value when scrolled into viewport. Uses requestAnimationFrame for smooth easing.

IntersectionObserver trigger duration: 1200ms easing: easeOutExpo format: add '+' suffix at end
⌨️

Terminal Typewriter

Hero tagline or terminal components type out character by character. Blinking cursor shown before and after completion.

setInterval: 45ms/char cursor: blink 1s step-end ∞ delay: 400ms after page load erase: optional loop mode
🎯

Button Micro-interaction

Primary buttons scale down 2% on click (active state). Release triggers a ripple emanating from click point.

active: scale(0.98) transition: 100ms ripple: CSS pseudo-element
📜

Marquee / Ticker

Tech stack logos or partner names scroll horizontally in an infinite loop on the home page. Pauses on hover.

@keyframes marquee translateX(0) → translateX(-50%) duration: 30s linear infinite pause-on-hover via JS
🌊

Page Transitions

SPA-style fade between page sections. New content slides up while old content fades out — 250ms total transition.

outgoing: opacity 0 + translateY(-8px) incoming: opacity 1 + translateY(0) duration: 250ms ease overlap: 50ms

CHAPTER 05 UX Improvement Strategy 15 strategic UX improvements over typical college club websites
05 · 01

Strategic UX Improvements

Each improvement is grounded in a real pain point of college club website visitors — students, potential members, faculty, and sponsors.

UX Problem #1 → Solution: No Visible CTA in Hero Most club sites bury their primary action. Fix: "Join Community" is the most prominent element in the hero — pill button, high contrast, above the fold on all devices.
UX Problem #2 → Solution: Events Are Hard to Find Users scroll endlessly to find next event. Fix: Featured event with live countdown timer at the top of Events page + sticky filter bar. Zero ambiguity.
UX Problem #3 → Solution: No Proof of Activity Visitors can't tell if the club is active or dead. Fix: Live GitHub stats widget, real-time event countdown, recent project commits in terminal widget.
UX Problem #4 → Solution: Resources Are Inaccessible Resources hidden behind Google Drive links. Fix: Searchable resource library with live filtering — no clicks needed to find what you're looking for.
UX Problem #5 → Solution: No Domain Clarity Visitors don't understand what domains or tracks exist. Fix: Domain tabs on Teams page + Tech Stack ticker on Home + domain-coded project and resource cards.
UX Problem #6 → Solution: Mobile Navigation Breaks Most club navbars are unusable on mobile. Fix: Full-height slide-in drawer on mobile, large touch targets, all 6 links + CTA visible without scrolling.
UX Problem #7 → Solution: No Conversion Path for "Interested But Not Ready" Binary join or leave. Fix: Three distinct join paths — Member (low friction), Core Team (selective), Propose a Talk (contribution-based). Meets people where they are.
UX Problem #8 → Solution: Team Page Has No Hierarchy Photo grids with no structure. Fix: Lead cards are visually larger + prominently placed. Domain tabs organize member cards. Clear seniority signals via card size.
UX Problem #9 → Solution: Past Events Are Lost No archive or recap. Fix: Past events section with "View Recap" links. Photo gallery links, report PDFs, and video recordings attached per event.
UX Problem #10 → Solution: No Proof of Real Work No project showcases. Fix: Dedicated Projects page with rich cards, live demo links, GitHub stats — this is the #1 trust builder for potential members and sponsors.
UX Problem #11 → Solution: Slow Initial Load Impression Typical sites front-load images and large assets. Fix: Dark hero with CSS-only grid background and text animation — fast first contentful paint. Images lazy-load below the fold.
UX Problem #12 → Solution: No Personality or Voice Generic corporate tone. Fix: Micro-copy with developer humor. Terminal command labels. Section names like "Meet the Builders" instead of "Our Team."
UX Problem #13 → Solution: No Social Proof Just numbers. Fix: Real testimonial cards with photos and LinkedIn badges. Authentic quotes from actual members about specific outcomes (not just "great club!").
UX Problem #14 → Solution: No Alumni Connection Alumni are invisible after graduation. Fix: Alumni strip on Teams page showing past chapter leads + their current roles. Aspirational signal for current members.
UX Problem #15 → Solution: No Contribution Pathways Community feels one-directional (receive only). Fix: Multiple contribute CTAs — submit a project, propose a talk, add a resource, apply for core team. Members as contributors, not just consumers.
